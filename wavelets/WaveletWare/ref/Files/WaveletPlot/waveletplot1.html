Iteration  (5)Iteration  (5)

The option iter can be set to highlight particular portions of the wavelet transform.

If a is a matrix, then iter is a list of ordered triples. If tree is a list of three trees, then iter should be a list of 4-tuples. If tree is a single tree, then iter should be a list of ordered triples.

If a is matrix or a is a list of three matrices and only one tree is given, then the elements of iter are of the form where i is the iteration, and is the block of the desired region of the transformation. If a is a list of three matrices, then the elements of iter are of the form where c is the channel 1,2,3, i is the iteration and is the block of the desired transformation.

In the case when a is a list of matrices and tree is a list of three trees, an element of iter can be {All,i,j,k}. In this case, the regions are extracted where c=1,2,3.

PlotTogether  (5)PlotTogether  (5)

In certain cases, WaveletPlot can be instructed to create a color plot from the transformed data. Such cases are color wavelet transforms when Iteration is set to All or the same regions from each channel are selected, a wavelet packet transform that was computed using the CompositeTree option set to True, or a wavelet packet transform where the same region(s) is(are) picked from each channel. WaveletPlot does its best to determine if matrices can be combined when PlotTogether is set to True, and if it is not possible, resets PlotTogether to False.

DimmingRange  (2)DimmingRange  (2)

DimmingRange is the value assigned to IntensityRange in the MatrixStretch function. All dimmed portions of the transform are scaled by default. The default value of DimmingRange is {200,255} which scales the region in a lighter tone of the dimming color. DimmingRange must be of the form {a,b}, where a ⩽ b are integers in the range 0,...,255.

Use DimmingRange to set the scaling used by plotted regions of the transform.
