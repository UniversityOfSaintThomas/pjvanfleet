Computation  (1)Computation  (1)

By default, all computations of the wavelet transform are numerical. However, if the input data are symbolic, the filter values are symbolic and Computation is set to Symbolic, then the computations are done symbolically. Note that if the filter is given in numerical format, then the computations are done numerically.

Create integer-valued data and integer-valued noise so that the wavelet transform can be computed symbolically.

Shift  (1)Shift  (1)

The default value for Shift is 0. This means the orthogonal filter processes the data starting with the first entry and continues, moving over by 2 each time. Setting the shift to a different integer changes the starting point. This option can be set for the wavelet transform used in WaveletRegionTake.

Boundary  (1)Boundary  (1)

For symmetric biorthogonal filter pairs, Boundary can be set to True and in this case, BWT exploits the symmetry of the filters to more accurately represent the transform at the beginning and end of the data set.

In the case where the filter lengths are both odd, BWT uses as input the (2n-2)-length vector

        (, ..., , , , ..., , )

and in cases where the filter lengths are both even, BWT uses as input the 2n-length vector

        (, ..., , , , ,..., )

and in both cases, takes the first n/2 values of the output for the lowpass portion and the first n/2 values of the last n-1 (filter lengths odd) or n (filter lengths even) components for the highpass portion to build the transform.

WaveletShrinkage can use biorthogonal filters and exploit the Boundary setting.

IntegerMap  (1)IntegerMap  (1)

The LeGall wavelet transform employs the lifting scheme to perform the computations and can, by setting IntegerMap to True, map integer-valued input to integer-valued output. This option can be set in WaveletShrinkage although we must ensure that the data and noise are integer-valued.
