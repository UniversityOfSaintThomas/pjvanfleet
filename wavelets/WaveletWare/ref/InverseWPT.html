InverseWPT


computes the inverse wavelet packet transform of a vector, a matrix or a list of three matrices.

DetailsDetails

  • The following options can be given:
  • Shift0Shift is an option for discrete wavelet transform functions to indicate if the lowpass/highpass filters should be shifted for the computation.
    BoundaryFalseBoundary is an option for several routines that either compute or use the biorthogonal wavelet transform. If set to True, the symmetric biorthogonal transformation is computed.
    IntegerMapFalseIntegerMap is an option used by LeGall wavelet transform functions. If set to True (default is False) and the input are integers, then the output is integer-valued.
    ComputationNumericalComputation is an option for wavelet transform functions or the CE function that indicates whether the comptuation should be done numerically or symbolically.
  • wt should be a vector, a matrix or a list of three matrices of equal dimensions.
  • tree should be a wavelet packet tree if wt is a vector or matrix. If wt is a list of three matrices of equal dimensions, then tree can be either a single wavelet packet tree (this assumes CompositeTree has been set to True when using WPT to compute wt) or a list of three wavelet packet trees each of the same length.
  • h is the filter or filter pair used to create the packet transform.
  • The inverse wavelet packet transform is computed using information from the tree and the appropriate routine (either InverseHWT, InverseWT, InverseBWT or InverseLWT).
  • The options NumIterations, Shift, Boundary, IntegerMap and Computation are options for the inverse (bi)orthogonal wavelet transforms InverseHWT, InverseWT, InverseBWT and InverseLWT. See these help pages for more information.
