SUREShrink


takes a vector or matrix and a wavelet filter, an orthogonal filter or biorthogonal filter pair, and returns a denoised approximation of the input.

DetailsDetails

  • The following options can be given:
  • NumIterations1NumIterations is an option used by several functions in the WaveletWare package. It indicates the number of iterations of the wavelet transform performed on the data.
    ThresholdByLevelFalseThresholdByLevel is a flag that tells VisuShrink and to use threshold values for each individual iteration of the wavelet transform instead of the default single value.
    ComputationNumericalComputation is an option for wavelet transform functions or the CE function that indicates whether the comptuation should be done numerically or symbolically.
    Shift0Shift is an option for discrete wavelet transform functions to indicate if the lowpass/highpass filters should be shifted for the computation.
    BoundaryFalseBoundary is an option for several routines that either compute or use the biorthogonal wavelet transform. If set to True, a symmetric biorthogonal transformation is computed.
    IntegerMapFalseIntegerMap is an option used by LeGall wavelet transform functions. If set to True (default is False) and the input are integers, then the output is integer-valued.
  • All options, except for ThresholdByLevel, are for (bi)orthogonal wavelet transform functions.
  • The SURE Shrink algorithm is as follows:
  • Compute k iterations of the (bi)orthogonal wavelet transform of data.
    Compute the DonohoSURE threshold (vector) λ.
    Check the sparseness of the highpass portion (or each highpass portion). If the part is sparse, use the UniversalThreshold tolerance, else use the DonohoSURE tolerance in conjunction with the shrinkage function to threshold the part.
    Join the new highpass portions with the lowpass portion of the transform and compute the inverse transform to obtain the denoised data.
