WPT


computes the wavelet packet transform of a vector, a matrix or a list of three matrices, returning the transform and the packet tree.

DetailsDetails

  • The following options can be given:
  • NumIterations1NumIterations is an option used by several functions in the WaveletWare package. It indicates the number of iterations of the wavelet transform performed on the data.
    Shift0Shift is an option for discrete wavelet transform functions to indicate if the lowpass/highpass filters should be shifted for the computation.
    BoundaryFalseBoundary is an option for several routines that either compute or use the biorthogonal wavelet transform. If set to True, the symmetric biorthogonal transformation is computed.
    IntegerMapFalseIntegerMap is an option used by LeGall wavelet transform functions. If set to True (default is False) and the input are integers, then the output is integer-valued.
    ComputationNumericalComputation is an option for wavelet transform functions or the CE function that indicates whether the comptuation should be done numerically or symbolically.
    SecondParameterNoneSecondParameter is an option for BestBasisTree and that allows a user to input a cost function that has a second parameter.
    CostFunctionNumberNonzeroCostFunction is an option that allows the cost function to be set to something different than NumberNonzero (default) in BestBasisTree and .
    CompositeTreeFalseCompositeTree is an option for that when set to True (default is False) and the input is a list of three matrices, creates a single best basis tree.
    FixedTree{}FixedTree is an option for that directs to used a fixed basis tree rather than generate a best basis tree.
  • The wavelet packet transform is computed by first calculating the full wavelet packet decomposition of data using WPTFull. The output from this computation is used by BestBasisTree to determine the best basis for the transform.
  • The options NumIterations, Shift, Boundary, IntegerMap and Computation are options for the (bi)orthogonal wavelet transforms WT, HWT, BWT, LWT. See these help pages for more information.
  • The options CostFunction and SecondParameter are used for computing the best basis tree. See the documentation for these options and BestBasisTree for more information.
  • For lists of three matrices, the option CompositeTree can be set to True so that WPT uses the same tree for each the full packet decomposition of each channel. To obtain the composite tree, at each level, the channels are averages and BestBasisTree is used on this composite matrix.
  • The best basis algorithm can be bypassed by setting FixedTree to a user-defined tree (or list of three trees). For example, WT[data,h,NumIterations->its] is equivalent to WPT[FixedTree->WaveletTree[its],NumIterations->its] when data is a vector. In the case where data is a matrix (or list of matrices), FixedTree should be set to WaveletTree[its,InputDimension->2].
