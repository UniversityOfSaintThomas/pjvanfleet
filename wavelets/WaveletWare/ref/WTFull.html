WTFull


takes a vector, a matrix or a list of three matrices and a (biorthogonal) filter and returns the (biorthogonal) wavelet transform for iterations 0,1,2,...

DetailsDetails

  • The following options can be given:
  • NumIterations1NumIterations is an option used by several functions in the WaveletWare package. It indicates the number of iterations of the wavelet transform performed on the data.
    Shift0Shift is an option for discrete wavelet transform functions to indicate if the lowpass/highpass filters should be shifted for the computation.
    BoundaryFalseBoundary is an option for several routines that either compute or use the biorthogonal wavelet transform. If set to True, the symmetric biorthogonal transformation is computed.
    IntegerMapFalseIntegerMap is an option used by LeGall wavelet transform functions. If set to True (default is False) and the input are integers, then the output is integer-valued.
    ComputationNumericalComputation is an option for wavelet transform functions or the CE function that indicates whether the comptuation should be done numerically or symbolically.
  • h can be an orthogonal filter or a biorthogonal filter pair.
  • The output is returned as a list of vectors or matrices or lists of three matrices of equal dimensions with the first being data, followed by wavelet transformations of data with 1,2,...,NumIterations iterations.
  • All options are for (bi)orthogonal wavelet transformations. See help for WT, HWT, BWT, or LWT for more information.
  • A second option for Computation is Symbolic.
  • Symbolic computation will be overridden if either the input filter or the data are given in numerical form.
