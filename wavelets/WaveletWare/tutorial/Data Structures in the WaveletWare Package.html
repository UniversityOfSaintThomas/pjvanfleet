Input for Computing and Plotting

(top)

Routines that compute (biorthogonal) wavelet (packet) transformations expect input data in one of the following formats: a vector (list), a matrix or a list of three matrices each of which has the same dimensions.

The function CheckData can be used (and is used internally by many WaveletWare functions) to determine if input data is in one of the correct formats.

CheckDataDetermines format of data

Checking data format.

CheckData always returns a list of length 2. The first number is 1,2,3,4 depending on if the input data is a vector, matrix, list of three matrices of equal dimensions, or none of the above. The second value is the maximum number of iterations that can be performed on the input data.

Trees in WaveletWare

(top)

A tree in the WaveletWare package is a structure that indicates a basis decomposition of a wavelet packet transformation. Since a wavelet decomposition is a wavelet packet decomposition, a tree can be defined for a wavelet transform as well.

All values in a tree are either 0 or 1. A 0/1 indicates that a certain portion of the transform is not/is part of the decomposition.

(top of section)

Then counting the original data, there are k+1 vectors in the output (with the original data counting as the 0th iteration). At level k, the data are decomposed into blocks, each of equal length. The Iteration Option

(top)

In visualization routines such as WaveletVectorPlot, WaveletPlot, FullWaveletVectorPlot and FullWaveletPlot, the Iteration option can be set to indicate particular portions of the transformed data to be plotted.

WaveletVectorPlot and FullWaveletVectorPlot

(top of section)

To view a particular region or regions of a wavelet transformation, the Iteration option can be set. In the case of WaveletVectorPlot, the value for Iteration is a list of lists each of length two. If {i,j} is a particular list element, then i is the iteration for the region to be plotted and j=1,...,. Note that {i,j} must represent a valid portion of the transform data.

FullWaveletVectorPlot plots data generated by WTFull or WPTFull. The function WTFull creates wavelet transforms for a prescribed number of iterations i (so each level is an iterated transform) and WPTFull generates a full decomposition at each level. While regions in the output of WPTFull could be identified with lists of length 2, regions in the output of WTFull need lists of length 3. Thus for FullWaveletVectorPlot, the Iteration option value is a list of lists each of length three.

If a particular list element is {i,j,k}, then i is the level of the output and {j,k} is the iteration and region as described for WaveletVectorPlot.

WaveletPlot and FullWaveletPlot

(top of section)

For matrix output plotted by WaveletPlot, the Iteration option can be set to highlight only particular regions of the plot. In this case the value of Iteration is a list of lists of length 3. If {i,j,k} is a list element, then i is the iteration of the region and then {j,k} represents the block location of the region.

When FullWaveletPlot is used to plot output from WTFull or WPTFull, the value for the Iteration option depends on the number of iterations that were computed of the transform. If n iterations were computed, then the value of Iteration is a list of length n+1. Each list is then itself a list of ordered triples where the triples are described in the above section. The value All can be used to plot all regions at a particular level and {} can be used to indicated no regions at the particular level are to be plotted.

WaveletPlot and FullWaveletPlot with Color Images

(top of section)

For color images with WaveletPlot, the value of Iteration is a list of lists each of length 4. If {c,i,j,k} is a list element, then c is the channel to be plotted and {i,j,k} is as described for a list value for Iteration when plotting matrices (see above). In the case where the plotted data is from a wavelet transform or the regions in a packet transform are to all be plotted, the value All can be substituted for c.

For color images with FullWaveletPlot, the value of Iteration depends on the number of iterations n of the transform that have been computed. In this case, the value of Iteration is a list of length n+1 and each element in this list is as described for color images and WaveletPlot in the above section. All and {} can be used in lieu of a list item to plot all or none of the regions at a particular level.
